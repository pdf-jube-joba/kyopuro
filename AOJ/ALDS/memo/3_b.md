# Queue

## 内容
ラウンドロビンスケジューリングを行う。

## メモ
rust にも queue は vecdeque があるのだが、趣旨としてはこれに似たものを実装してみようということだと考えたので、自分でいろいろ書いた。

- 普通に vec を使い、 push と pop では vec を新たに作り直す/ vec にある push() や pop() を使う
    - どちらを頭にするにせよ、一方は $O(n)$ の操作になる。
    これは vec を一つ除いてコピーする必要があるため
- ラウンドロビンの図を見たので、それを実装しようと考えた。

条件を見ると、キューの長さが最初に作った時を超えることはないので、これを用いる。
vec に加えて、 どこからどこまでに値が入っているかを入れておく値を入れた。

```
       h     t
[_, _, 1, 2, 3, _]
```
のようなイメージ。
- キューが空の場合を除き head, tail は値が入っているところをさしている。
- キューの長さは固定として、操作するときは Result や Option を返す。

## 改善点
`is_empty` はいらないと思う。
mod 演算を関数にするよりも、 head+1,head-1などをメソッドにするほうがよい？
head や tail をうまく設定することで empty かどうかによらない議論ができるのではないか？
あと、データ構造の不変条件の一つに、 Some や None は必ず連続しているというところがあるので、それを使うと簡潔に書けそう。
